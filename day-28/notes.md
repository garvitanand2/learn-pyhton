# Day 28 Notes — Clean Python: Type Hints & Dataclasses

## Why Type Annotations?

Python is dynamically typed — variables have no declared type. Type hints are **optional annotations** that:
- Act as inline documentation (readable without running the code)
- Enable static analysis tools (`mypy`, `pyright`) to catch bugs before execution
- Improve IDE autocomplete and refactoring
- Zero runtime cost (ignored by the Python interpreter by default)

```python
# Before — what does this accept? Returns what?
def process(data, k):
    ...

# After — crystal clear
def process(data: list[str], k: int = 5) -> dict[str, float]:
    ...
```

## Basic Type Hints Reference

| Annotation | Meaning |
|-----------|---------|
| `x: int` | integer |
| `x: float` | float |
| `x: str` | string |
| `x: bool` | boolean |
| `x: None` | only None |
| `x: list[str]` | list of strings |
| `x: dict[str, int]` | dict with str keys, int values |
| `x: tuple[int, str]` | fixed-length tuple |
| `x: set[str]` | set of strings |
| `x: Optional[str]` | str or None |
| `x: Union[str, int]` | str or int |
| `x: Any` | any type (escape hatch) |
| `x: Final[int]` | constant |

> **Python 3.10+ shorthand:** `str | None` instead of `Optional[str]`

## Optional vs Union

```python
from typing import Optional, Union

# These are equivalent:
def fn1(x: Optional[str]) -> None: ...
def fn2(x: Union[str, None]) -> None: ...
def fn3(x: str | None) -> None: ...   # Python 3.10+

# Use Optional when something CAN be absent (not provided / missing data)
def find_user(user_id: int) -> Optional[str]:
    return db.get(user_id)   # None if not found
```

## TypeVar — Generic Functions

```python
from typing import TypeVar
T = TypeVar("T")

def first(items: list[T]) -> T:
    return items[0]

first([1, 2, 3])       # inferred return type: int
first(["a", "b"])      # inferred return type: str
```

## @dataclass — Complete Reference

```python
from dataclasses import dataclass, field, asdict, astuple

@dataclass
class Config:
    name: str                                    # required field
    lr: float = 1e-3                             # field with default
    tags: list[str] = field(default_factory=list) # mutable default
```

### Auto-generated methods
| Method | Generated by default | Notes |
|--------|---------------------|-------|
| `__init__` | yes | all fields become parameters |
| `__repr__` | yes | pretty print |
| `__eq__` | yes | compares all fields |
| `__hash__` | **no** (unless `frozen=True`) | |
| `__lt__`, `__gt__` | only if `order=True` | |

### Key Parameters
```python
@dataclass(frozen=True)   # immutable — raises FrozenInstanceError on mutation
@dataclass(eq=False)      # don't generate __eq__
@dataclass(order=True)    # generate __lt__, __le__, __gt__, __ge__
@dataclass(slots=True)    # Python 3.10+ — uses __slots__ for memory efficiency
```

### `__post_init__` — Validation Hook
```python
@dataclass
class Config:
    lr: float = 1e-3

    def __post_init__(self):
        if self.lr <= 0:
            raise ValueError(f"lr must be positive, got {self.lr}")
```

### `field()` Options
```python
field(default=0)                    # same as = 0
field(default_factory=list)         # callable to create mutable default
field(repr=False)                   # exclude from __repr__
field(compare=False)                # exclude from __eq__
field(init=False)                   # not a parameter in __init__
```

### Serialization
```python
from dataclasses import asdict, astuple
asdict(cfg)    # → dict  — easy JSON export: json.dumps(asdict(cfg))
astuple(cfg)   # → tuple — lightweight for CSV/tabular output
```

## Protocol — Structural Typing

```python
from typing import Protocol

class Scorer(Protocol):
    def score(self, text: str) -> float: ...

# No inheritance required — any class with .score(text) → float satisfies it
class RuleScorer:
    def score(self, text: str) -> float:
        return 1.0 if "good" in text else 0.0

def rank(scorer: Scorer, texts: list[str]) -> list[tuple[str, float]]:
    scored = [(t, scorer.score(t)) for t in texts]
    return sorted(scored, key=lambda x: x[1], reverse=True)
```

**Protocol vs ABC:**
| | Protocol | ABC |
|---|---------|-----|
| Checks | structural (duck typing) | nominal (must subclass) |
| Inheritance | not needed | required |
| Runtime check | no (static only) | yes (`isinstance`) |
| Use case | external classes / frameworks | your own class hierarchy |

## TypedDict

```python
from typing import TypedDict

class Metrics(TypedDict):
    accuracy: float
    f1: float

# Works exactly like a dict at runtime — just annotated
m: Metrics = {"accuracy": 0.92, "f1": 0.89}
```

## Final — Constants

```python
from typing import Final

MAX_LEN: Final[int] = 512   # mypy will flag: MAX_LEN = 1024

# In a class — equivalent to a class constant
class Config:
    VERSION: Final = "1.0.0"
```

## Running mypy

```bash
pip install mypy
mypy app.py            # check one file
mypy .                 # check entire project
mypy --strict app.py   # strictest mode
```

Common mypy errors:
- `Argument 1 to "fn" has incompatible type "int"; expected "str"`
- `Item "None" of "Optional[str]" has no attribute "upper"`
- `Cannot assign to final name "MAX_LEN"`
